Action Situation 1: Capacitor Adoption Assurance Game  
Tension: Two farmers sharing a transformer must decide whether to adopt voltage‑stabilising capacitors. Mutual adoption yields shared improvement, but unilateral adoption brings no private benefit and wastes the investment, making coordinated adoption risky.  
Matrix (normal‑form, ordinal payoffs: 4 = best, 1 = worst):  
```
          Farmer 2
          Adopt   Not
Farmer 1 Adopt  (3,3)  (1,4)
         Not    (4,1)  (2,2)
```  
Justification: ODD describes AS1 as a capacitor‑adoption assurance game where “mutual investment yields shared improvement, while unilateral investment yields no added private benefit” (III.iv.a).

Action Situation 2: Sequential Social‑Learning in Capacitor Adoption  
Tension: A farmer observes a peer’s capacitor‑adoption outcome and imitates only if that outcome ranks higher; diffusion therefore depends on visible success of prior coordinated trials.  
Sequential Representation (game tree):  
```
Player 1 chooses Adopt (A) or Not (N).
Player 2 observes Player 1’s action and chooses A or N.

Payoffs (Player 1, Player 2):
- If Player 1 A:
    - Player 2 A: (3,3)
    - Player 2 N: (1,4)
- If Player 1 N:
    - Player 2 A: (4,1)
    - Player 2 N: (2,2)
```  
Justification: ODD defines AS2 as “a sequential social‑learning process in capacitor adoption in which each farmer observes a peer’s outcome and imitates only if that outcome ranks higher” (III.iv.a).

Action Situation 3: Asymmetric Transformer‑Capacity Authorization Dilemma  
Tension: One farmer’s investment in transformer capacity benefits both, but the cost is borne solely by the investor, creating a free‑rider incentive.  
Matrix (normal‑form):  
```
          Farmer 2
          Invest  Not
Farmer 1 Invest (3,3)  (1,4)
         Not    (4,1)  (2,2)
```  
Justification: ODD states AS3 is “an asymmetric transformer‑capacity authorization dilemma … if only one invests, the contributor bears cost while the non‑investor benefits more” (III.iv.a).

Action Situation 4: Mutual‑Exchange Coordination Game  
Tension: A farmer and sub‑station staff must coordinate on informal exchange; mutual engagement yields reciprocal benefit, but mismatched expectations cause losses.  
Matrix (normal‑form):  
```
          Staff
          Engage  Abstain
Farmer  Engage (3,3)   (1,2)
        Abstain (2,1)   (2,2)
```  
Justification: ODD describes AS4 as “a mutual‑exchange coordination game … reciprocal benefit arises only when both engage; if either abstains while the other offers, the offerer bears a loss while the abstainer reverts to baseline” (III.iv.a).

Action Situation 5: Authorization‑and‑Investment Asymmetric Coordination  
Tension: A farmer chooses between formal and informal electricity access while staff decide whether to invest in capacity; formal cooperation is collectively optimal but burden‑sharing is asymmetric.  
Matrix (normal‑form):  
```
          Staff
          Invest  Withhold
Farmer  Formal   (3,2)    (1,3)
        Informal (4,1)    (2,2)
```  
Justification: ODD defines AS5 as an “authorization‑and‑investment asymmetric coordination game … mutual formal cooperation is collectively optimal, but … staff gain modestly even under formal cooperation due to investment burden” (III.iv.a).

Action Situation 6: Groundwater Extraction Prisoner’s Dilemma  
Tension: Two farmers drawing from the same aquifer must choose between restraint and over‑extraction; individual over‑extraction is tempting but mutual over‑extraction accelerates depletion.  
Matrix (normal‑form):  
```
          Farmer 2
          Restrain Extract
Farmer 1 Restrain (3,3)    (1,4)
         Extract  (4,1)    (2,2)
```  
Justification: ODD describes AS6 as “a groundwater‑extraction prisoner’s dilemma … mutual restraint sustains yields but unilateral over‑extraction offers short‑term gain” (III.iv.a).
